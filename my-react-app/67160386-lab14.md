### 📝 คำถาม 1.2: Route แบบ Dynamic ทำงานอย่างไร? และ useParams ดึงข้อมูลอะไร?

```
Dynamic Route ทำงานอย่างไร?
Static Route จะเป็นการจับคู่ URL ตรงๆ เช่น /about หรือ /contact แต่ Dynamic Route จะใช้ "Path Parameters" เป็นตัวแปรเพื่อบอก Router ว่า "ตรงนี้คือข้อมูลที่จะเปลี่ยนไปตามสถานะนะ"

สัญลักษณ์สำคัญ: เราจะใช้เครื่องหมาย Colon (:) นำหน้าชื่อตัวแปรในตอนกำหนด Route

หลักการ: เมื่อ URL ใน Browser ตรงกับ Pattern ที่เราวางไว้ Router จะทำการ "จับคู่" (Matching) และเก็บค่าในตำแหน่งนั้นไว้เพื่อให้เรานำไปใช้งานต่อ
```

**ตัวอย่างการกำหนด Route:**

```jsx
<Route path="/product/:id" element={<ProductDetail />} />
```

- ถ้า URL คือ /product/123 -> ค่าของ id จะเท่ากับ 123

- ถ้า URL คือ /product/iphone -> ค่าของ id จะเท่ากับ iphone

```text
useParams ดึงข้อมูลอะไร?
useParams คือ Custom Hook ที่มากับ React Router ซึ่งทำหน้าที่ดึงค่าจาก URL ที่เรากำหนดไว้ใน Path Parameters มาอยู่ในรูปแบบของ Object

ข้อมูลที่ดึงมา: มันจะดึงเอาค่า (Value) ที่อยู่บน URL ณ ตำแหน่งที่เราใส่ : ไว้

รูปแบบผลลัพธ์: จะได้เป็น Object ที่มี Key ตรงกับชื่อที่เราตั้งใน Route และ Value ตรงกับสิ่งที่ปรากฏบน URL
```

ตัวอย่างการใช้งาน

- ขั้นแรก: กำหนด Route

```jsx
// App.js
<Route path="/user/:username" element={<UserProfile />} />
```

- ขั้นสอง: ดึงข้อมูลมาใช้ใน Component

```jsx
// UserProfile.js
import { useParams } from "react-router-dom";

function UserProfile() {
  const { username } = useParams(); // ดึงค่าจาก URL มาเก็บในตัวแปร username

  return (
    <div>
      <h1>ยินดีต้อนรับคุณ: {username}</h1>
    </div>
  );
}
```

### 📝 คำถาม 1.3: useNavigate ใช้เพื่ออะไร? และ navigate(-1) หมายความว่าอะไร?

**ใช้เพื่ออะไร**

```text
เป็น Hook ที่คืนค่าเป็น ฟังก์ชัน สำหรับสั่งให้ Browser เปลี่ยนไปยัง URL ที่เราต้องการ มักใช้ในกรณีที่ต้องมีการทำงานบางอย่างให้เสร็จก่อนแล้วค่อยเปลี่ยนหน้า (Logic-based navigation)
```

**ตัวอย่าง**

```jsx
import { useNavigate } from "react-router-dom";

function LoginPage() {
  const navigate = useNavigate();

  const handleLogin = () => {
    // สมมติว่าตรวจสอบ Login สำเร็จ
    navigate("/home"); // สั่งให้ไปหน้า Home ทันที
  };

  return <button onClick={handleLogin}>Log In</button>;
}
```

**navigate(-1) หมายความว่าอะไร?**

```text
คำสั่ง Maps(-1) คือการสั่งให้ Browser "ย้อนกลับไป 1 ลำดับ" (Go Back) ในประวัติการเข้าชม (History Stack) ของผู้ใช้งาน

หลักการ: เหมือนกับการที่ผู้ใช้กดปุ่ม "Back" บน Browser

ตัวเลขติดลบ: เลข -1 หมายถึงย้อนกลับไป 1 หน้า ถ้าใส่ -2 ก็คือย้อนกลับไป 2 หน้า

ตัวเลขเป็นบวก: หากใส่ Maps(1) จะเป็นการกด "Forward" (ไปข้างหน้า) หากผู้ใช้เคยย้อนกลับมาแล้ว
```

```text
ตัวอย่าง
- Maps('/about'):	เปลี่ยนไปยังหน้า /about
- Maps(-1):	ย้อนกลับไปหน้าก่อนหน้า (Back)
- Maps(1):	ไปหน้าถัดไป (Forward)
- Maps('/', { replace: true }):	ไปหน้าแรก โดย ไม่เก็บหน้าปัจจุบัน ไว้ใน History (ย้อนกลับมาไม่ได้)
```

### 📝 คำถาม 2.1: เหตุใดจึงต้องใช้ Context แทนการส่ง Props ในแอปที่มีหลายหน้า? ยกตัวอย่างข้อมูลที่เหมาะกับ Context

```
Prop Drilling คืออะไร?: คือการที่เราต้องส่ง Props ผ่าน Component ลูกหลานไปเรื่อยๆ ทั้งที่ Component ตัวกลางเหล่านั้นไม่ได้ใช้งานข้อมูลนั้นเลย เพียงแค่ "เป็นทางผ่าน" ให้ส่งไปถึง Component ปลายทางเท่านั้น
```

```text
ข้อมูลที่นิยมใช้ Context:
- User Authenticationเพื่อให้ทุกหน้า (Profile, Settings, Navbar) รู้ว่าใครกำลัง Login อยู่
- Theme (Dark/Light)เพื่อให้ทุก UI Component เปลี่ยนสีตามโหมดที่ผู้ใช้เลือกพร้อมกัน
- Language / Localizationเพื่อให้ข้อความทั้งแอปเปลี่ยนเป็นภาษาไทยหรืออังกฤษตามการตั้งค่า
- Shopping Cartในหน้าสินค้าหรือหน้าชำระเงิน ต้องเข้าถึงรายการสินค้าในตะกร้าตัวเดียวกัน
```

### 📝 คำถาม 2.2: ถ้า Component ใช้ useContext แต่ไม่ได้อยู่ใน Provider จะเกิดอะไรขึ้น? วิธีป้องกัน?

**จะเกิดอะไรขึ้น?**

```text
โค้ดของคุณพยายามเข้าถึง Property ใน Object นั้น (เช่น user.name) แต่ค่าที่ได้ดันเป็น undefined แอปของคุณจะเกิด Error หน้าจอขาวทันที (TypeError: Cannot read properties of undefined).
```

**วิธีป้องกัน**

- กำหนด Default Value ให้ชัดเจน

```jsx
const ThemeContext = createContext("light"); // ถ้าหา Provider ไม่เจอ จะได้ค่า 'light' เสมอ
```

- สร้าง Custom Hook พร้อมตัว Check (แนะนำ)
  - วิธีที่เป็นมาตรฐานคือการสร้าง Hook ขึ้นมาใช้เอง และสั่งให้มัน Throw Error ออกมาทันทีถ้าอยู่นอกเขต Provider

```jsx
// ในไฟล์ Context ของคุณ
export function useMyContext() {
  const context = useContext(MyContext);

  // ถ้า context เป็น undefined หรือ null (กรณีไม่ได้ตั้ง default หรือตั้งไว้เป็น null)
  if (!context) {
    throw new Error("useMyContext must be used within a MyProvider");
  }

  return context;
}
```

## 📋 ส่วนที่ 3: Deployment

### 📝 คำถาม 3.1: ทำไม React Router ถึงต้องมีการ configure เพิ่มเติมบน Vercel/Netlify? อธิบายหลักการ

```text
React Router ใช้ client-side routing:
- URL เปลี่ยนโดยไม่มีการ request ไปที่ server
- เมื่อ refresh หน้า → browser ส่ง request ไปที่ server
- Server หา /products/1 ไม่เจอ → 404

วิธีแก้:
- ให้ server redirect ทุก request กลับมาที่ index.html
- React Router จะจัดการ routing เอง (vercel.json, _redirects)
```

**วิธีการ Configure**

- สำหรับ Vercel

```json
{
  "rewrites": [
    {
      "source": "/(.*)",
      "destination": "/index.html"
    }
  ]
}
```

- สำหรับ Netlify

```Plaintext
/* /index.html   200
```

(ความหมาย: ทุกเส้นทาง /\* ให้ส่งไปที่ /index.html ด้วย Status Code 200)
